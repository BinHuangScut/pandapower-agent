# Maintenance Cadence (Lightweight)

## Weekly cadence

- 1-2 maintainer triage sessions per week.
- Prioritize bugs and onboarding blockers first.
- Keep at least 5 issues labeled `good first issue`.

## Response SLA

- High-quality bug reports: first response within 48-72 hours.
- Security reports: acknowledge within 72 hours.

## Weekly update format

Publish one short update each week:

- New capabilities
- Important fixes
- Open contribution opportunities
